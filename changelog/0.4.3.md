# v0.4.3 — corpus CI jobs (informational)

Two new CI jobs track parse and round-trip coverage across the 18-package
corpus. Both run on every push and pull request but are informational only
(continue-on-error: true). They become hard gates in v0.4.8.

## corpus-parse

Restores corpus/src from the Actions cache (keyed by corpus/packages.txt hash),
builds gopapy, and runs `corpus/check.sh`. Reports pass and fail counts. At the
time of this release: 8999 passed, 7 failed (all intentionally invalid files
that CPython itself rejects).

## corpus-roundtrip

Same cache logic, then runs `corpus/roundtrip.sh` (gopapy unparse --check).
Reports round-trip pass and fail counts. Failures in this job reflect either
parser gaps in the legacy unparser or formatting differences; they do not
represent parse failures (those are tracked by corpus-parse).

## Cache strategy

Both jobs share the same cache key:
`corpus-src-${{ hashFiles('corpus/packages.txt') }}`. When packages.txt does
not change, corpus/src is restored from cache and the download step is skipped.
If both jobs run in the same workflow run, whichever populates the cache first
wins; the other job's save is a no-op.
