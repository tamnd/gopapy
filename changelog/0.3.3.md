## [0.3.3] - 2026-04-28

Phase 1.5, Round 3 of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

This round runs `gopapy check` and `gopapy unparse --check` against the
Python 3.9 through 3.13 standard libraries, fixes the failures found,
and wires up a proper per-version oracle test package.

### Fixed

- **Form feed (`\f`) and vertical tab (`\v`) in source files.** CPython's
  tokenizer treats `\f` and `\v` as horizontal whitespace. Files in the
  Python 3.9 and 3.10 email package use `\f` as a page separator on its
  own line; our scanner was rejecting these with "unexpected character".
  Fixed in both the new parser (`parser/lex.go`) and the legacy parser
  (`lex/scanner.go`, `lex/indent.go`) so both `gopapy check` and
  `gopapy unparse --check` handle these files. All 17 affected email
  package files now pass for Python 3.9 and 3.10.

- **Grammar fixture version markers.** Fixtures 059, 060, 061, 063, 070
  require Python 3.14+ syntax (t-strings, PEP 758) but had no `# Python
  3.14+` comment on the first line. Added the comment so `tests/run.sh`
  skips them gracefully under Python 3.13 and earlier. Fixes 5 spurious
  `ORACLE-ERR` failures under Python 3.13.

- **`tests/run.sh` version-skip logic.** Added a check for `# Python 3.X+`
  on the first line of each fixture. If the running Python is older than
  the required version, the fixture is reported as `SKIP` rather than
  `ORACLE-ERR`. Also added a note that the oracle diff is only meaningful
  on Python 3.13+ (where `show_empty=False` matches our format); on
  Python 3.12 and earlier the script still runs `gopapy dump` to check
  for crashes but does not compare the output.

### Added

- **`tests/compat/compat_test.go` — per-version oracle test package.**
  A Go test package that runs the grammar fixture oracle comparison for
  each Python minor version. It uses `uv python find 3.X` to locate the
  Python binary (falls back to `python3.X` on PATH). Features:
  - `TestOracle_Py313` — fully active, uses Python 3.13 via uv; 101 pass,
    5 skip (3.14+ fixtures), 0 fail.
  - `TestOracle_Py312` through `TestOracle_Py38` — placeholder tests with
    `t.Skip()` until version-aware dump is implemented (spec 1323-1324).
    Removing the `t.Skip()` is the only change needed to activate each
    version once `gopapy dump --py 3.X` produces correct output.

- **`Dockerfile.compat` + `scripts/compat_test.sh`.** A Docker image
  based on `golang:1.26-bookworm` with `uv` installed and Python 3.9
  through 3.13 pre-installed. `scripts/compat_test.sh` runs `gopapy
  check`, `gopapy unparse --check`, and `tests/run.sh` for each version.
  Lets developers reproduce the full CI compat matrix locally without
  needing every Python version installed.

- **`Makefile`.** `make compat-build` / `make compat-run` / `make compat`
  targets for the Docker compat workflow.

- **`compat-oracle` CI job.** New CI job with matrix `python-version ×
  os` (Python 3.8-3.13 × ubuntu/macos/windows). Each cell installs the
  target Python via `uv` and runs `go test ./tests/compat/...`. Python
  3.13 oracle is a passing hard gate; others skip gracefully until the
  version-aware dump is implemented.

- **`stdlib-compat` CI matrix expanded to `python-version × os`.**
  Added `macos-latest` as a second OS alongside `ubuntu-latest` so
  platform-specific parse differences surface in CI.
