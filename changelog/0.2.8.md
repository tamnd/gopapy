# 0.2.8 - 2026-04-27

### Changed

- **`lint` command** now runs entirely on the new `linter` package
  (parallel worker pool, filesystem cache, SARIF/JSON/text/github
  output) instead of the v1 `legacy/linter` orchestration. This is
  the lint half of the v2 swap that started with `dump` and `check`
  in 0.2.7.
- **`linter.LintFileWithConfig`** added so callers can lint a single
  file under a `linter.Config` without going through `LintFiles`.
  Wraps `LintFile` with code filtering (Select/Ignore/PerFile).
- **`linter` package decoupled from `legacy/`.** `linter/config.go`,
  `linter/cache.go`, `linter/format.go`, `linter/format_sarif.go`,
  and `linter/parallel.go` are now first-class members of the v2
  stack: they consume the new `diag.Diagnostic` (`Pos.Line`/`Pos.Col`)
  rather than the v1 shape (`Lineno`/`ColOffset`), and import nothing
  from `legacy/`. The new linter code path no longer depends on
  any v1 package.
- **No more `V2` suffix in the CLI.** `lintFileV2`/`lintFilesV2`
  wrapper functions were removed from `cmd/gopapy/main.go`; callers
  now use `linter.LintFileWithConfig` and
  `linter.LintFiles(paths, cfg, linter.LintOptions{Jobs, Cache})`
  directly.
- **CHANGELOG layout flattened to per-version files.** The single
  2300-line `CHANGELOG.md` is now an index that points at
  `changelog/x.y.z.md` files. Each release ships its notes in its
  own file, so future releases grow the directory rather than
  rewriting the index.
- CLI version bumped to `0.2.8`.

### Notes

- The `--fix` path still goes through the v1 fixers
  (`legacy/linter.FixWithConfig`, `cst.Parse`) because the safe
  rewrites operate on the v1 AST. A small `toLegacyConfig` adapter
  in `cmd/gopapy/main.go` converts `linter.Config` to
  `legacy/linter.Config` for that single path. Porting the fixers
  to the new AST is its own job and will arrive in a later release.
- The `lsp` server still uses `legacy/linter` end-to-end. Same
  reason: it depends on the v1 AST shape for go-to-def, and
  switching it is a separate workstream.
