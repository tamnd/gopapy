# v0.4.8 — corpus-parse and corpus-roundtrip hard gates

## Changes

- **corpus-parse hard gate**: `corpus/check.sh` now exits non-zero if any valid
  corpus file fails to parse. Seven intentionally-invalid black and Django test
  fixtures are skipped via `isIntentionalBadFixture` (they are rejected by
  CPython too). Result: **8999 passed, 0 failed**.

- **corpus-roundtrip hard gate**: `corpus/roundtrip.sh` now exits non-zero if
  any round-trip produces a mismatch or unexpected parse failure. Eight files
  with known legacy-parser limitations are passed via `--allow`; all other
  8991 files must round-trip cleanly. Result: **8999 files, 0 parse-failed,
  0 round-trip mismatched, 8 allowed**.

- **`isIntentionalBadFixture` extended**: Added seven corpus-specific base names
  for files that are intentionally invalid Python (both gopapy and CPython reject
  them): `async_as_identifier.py`, `invalid_header.py`,
  `pattern_matching_invalid.py`, `python2_detection.py`,
  `pep_572_do_not_remove_parens.py`, `pep_701.py`, `tests_syntax_error.py`.

- **CI**: Removed `continue-on-error: true` and `|| true` from both
  `corpus-parse` and `corpus-roundtrip` jobs. Both are now hard gates that
  block the build on regression.

## Known roundtrip allows (legacy parser limitations)

The 8 allowed files are tracked bugs in the legacy unparser, not the new parser:
- `pillow/TiffImagePlugin.py`, `pillow/WmfImagePlugin.py`,
  `black/python38.py` — annotated-assignment tuple RHS (`x: T = a, b`)
- `black/form_feeds.py` — form feed characters (U+000C) inside function bodies
- `black/type_param_defaults.py` — Python 3.13 `*Ts=int` type param default syntax
- `black/tricky_unicode_symbols.py` — NFKC identifier chars (deferred)
- `black/pep_572_remove_parens.py`, `black/preview_long_strings.py` — round-trip mismatches
