# 0.2.9 - 2026-04-27

Phase 1 kickoff of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

### Fixed

- **CI workflow.** `ci.yml` was running `go vet ./... ./v2/...`,
  `go build ./... ./v2/...`, etc. The `v2/` directory was removed
  in v0.2.6 so every CI run since then has failed with
  `pattern ./v2/...: lstat ./v2/: no such file or directory`. All
  five offending lines now use plain `./...`. CI on `main` is green
  again.
- **Async comprehensions.** The parser already supported `async for`
  inside the comprehension *clause loop*, but every comprehension
  *entry* check (`if p.isKeyword("for")`) only matched bare `for`.
  Constructs like `[x async for x in it]`, `{k: v async for k, v in
  it}`, `(x async for x in it)`, and the call-form genexp
  `f(x async for x in it)` all fell out of the comprehension branch
  and produced cryptic "expected ], got name" errors. Introduced
  `parser.isCompForStart()` and used it at all six sites
  (list / set / dict / parenthesised genexp / call-form positional /
  call-form general). Reproducer was `Lib/test/test_coroutines.py`.

### Added

- `tests/grammar/088_async_comp.py` covers list, set, dict,
  parenthesised genexp, multi-clause, `if`-filtered, and chained
  `async for` ... `async for` shapes.
- `~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md` is the
  written roadmap for closing the gap between "passes 87 fixtures
  + curated stdlib" and "parses every valid Python 3.14 program
  from PyPI". Includes honest current numbers: 1799 stdlib files
  pass, 43 fail. The 43 failures group into roughly six bug
  categories; Phase 1 is to drive that count to zero.

### Changed

- CLI version bumped to `0.2.9`.
