## [0.2.13] - 2026-04-28

### Fixed

- **`await` nested inside `await` in unparser.** `await (await bar())`
  is valid Python but `await await bar()` is a syntax error. The
  unparser was printing the inner `await` without parentheses, which
  made the round-trip test for `test/test_coroutines.py` produce
  invalid code. The `Await` case in `ast/unparse.go` now checks
  whether the value node is itself an `Await` and wraps it in
  parentheses when so.

### Added

- `tests/grammar/` — no new fixtures; the `test_coroutines.py`
  round-trip is covered by the stdlib unparse check in CI.

### Changed

- CLI version bumped to `0.2.13`.
