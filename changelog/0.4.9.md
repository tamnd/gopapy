# v0.4.9 — corpus-astdiff hard gate; Phase 2 complete

## Changes

- **corpus-astdiff hard gate**: `internal/oracle/corpus_diff.sh` now exits
  non-zero if any file produces an unexpected AST mismatch. Two known
  limitations are listed in `MISMATCH_ALLOW` and counted as `skip` rather than
  `fail`, keeping the gate strict for everything else.

- **`MISMATCH_ALLOW` in `corpus_diff.sh`**: Added allow list for:
  - `black/tricky_unicode_symbols.py` — NFKC identifier normalization (µ→μ)
    requires `golang.org/x/text`, deferred.
  - `pygments/unistring.py` — Go Unicode 15.0 vs Python Unicode 15.1 character
    category disagreements (e.g. U+1B4F, U+1B7F).

- **CI**: Removed `continue-on-error: true` and `|| true` from the
  `corpus-astdiff` job. It is now a hard gate that blocks on new regressions.
