## [0.3.5] - 2026-04-28

Phase 1.5, Round 5 of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

### Added

- **`TestOracle_Py312` — Python 3.12 oracle test (active, no skip).**
  `gopapy dump --py 3.12` is now verified byte-identical to CPython 3.12's
  `ast.dump()` for all compatible fixtures: 99 pass, 7 skipped (3.13+/3.14+
  syntax), 0 fail.

- **`showEmpty` path in `ASTDump` for Python ≤3.12.** Python 3.12 and earlier
  always print optional/empty fields that Python 3.13+ omits. The `dumper`
  struct's `showEmpty bool` flag (true when `pyMinor <= 12`) gates all these
  additions:
  - `Module`: `type_ignores=[]` always appended
  - `FunctionDef`/`AsyncFunctionDef`: `decorator_list=[]` always; `type_params=[]`
    when `pyMinor >= 12` (field didn't exist before 3.12)
  - `ClassDef`: `bases=[]`, `keywords=[]`, `decorator_list=[]` always;
    `type_params=[]` when `pyMinor >= 12`
  - `TypeAlias`: `type_params=[]` always (TypeAlias is itself 3.12+)
  - `arguments`: `posonlyargs=[]`, `args=[]`, `kwonlyargs=[]`, `kw_defaults=[]`,
    `defaults=[]` always
  - `Call`: `args=[]`, `keywords=[]` always
  - `comprehension`: `ifs=[]` always
  - `JoinedStr` (empty f-string): `values=[]` instead of bare `JoinedStr()`
  - `List`/`Tuple` (empty): `elts=[], ctx=...` instead of bare `ctx=...`
  - `For`/`AsyncFor`/`While`/`If`: `orelse=[]` always
  - `Try`/`TryStar`: `orelse=[]`, `finalbody=[]` always
  - `MatchSequence` (empty): `patterns=[]` instead of bare `MatchSequence()`
  - `MatchMapping` (empty keys/patterns): `keys=[], patterns=[]` shown
  - `MatchClass`: `patterns=[]`, `kwd_attrs=[]`, `kwd_patterns=[]` always

- **`# Python 3.12+` markers on 7 fixtures** that use Python 3.12+ syntax:
  - `054_type_alias.py` (type alias statement)
  - `055_type_param_func.py` (function type parameters)
  - `056_type_param_class.py` (class type parameters)
  - `062_fstring_pep701.py` (PEP 701 nested/multiline f-strings)
  - `093_fstring_tuple_expr.py` — updated `# parser: new` → descriptive
    comment (no Python version gate; valid on 3.8+)
  - `094_fstring_comment.py` — updated `# parser: new` → `# Python 3.12+`
  - `100_fstring_conv_whitespace.py` (PEP 701 conversion whitespace)
  - `101_fstring_selfdoc_comment.py` — updated `# parser: new` → `# Python 3.12+`

### Changed

- CLI version bumped to `0.3.5`.
