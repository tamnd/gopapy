## [0.3.2] - 2026-04-28

Phase 1.5, Round 2 of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

### Added

- **`parser/version.go` — VersionFeature table.** All knowledge of
  which grammar and dump features belong to which Python minor version
  now lives in one place. The type is a `uint32` bitmask so checking
  multiple features is a single AND operation and adding a new version
  is one line in the `versionFeatures` map. Each map entry is the full
  cumulative set for that minor version, not a delta, so reading any
  row gives the complete picture without tracing back through earlier
  entries. Features defined so far:
  - `FeatWalrus`, `FeatPosOnlyParams` (3.8+)
  - `FeatGeneralDeco` (3.9+, PEP 614)
  - `FeatMatchCase`, `FeatParenWith` (3.10+, PEP 634 / PEP 617)
  - `FeatExceptStar` (3.11+, PEP 654)
  - `FeatTypeParams`, `FeatTypeAlias`, `FeatPEP701FString` (3.12+, PEP 695 / PEP 701)
  - `FeatTypeParamDefaults` (3.13+, PEP 696)
  - `FeatTStrings` (3.14+, PEP 750)
- **`parser/version_test.go`.** Tests for `ParseVersion` (valid
  inputs, bad major, missing dot, garbage) and `FeaturesFor`
  (boundary versions, above-latest, below-3.8).
- **`gopapy dump --py 3.X` flag.** The CLI accepts `--py` (or
  `--python-version`) with a `3.X` string, parses it through
  `ParseVersion`, and looks up the feature set through `FeaturesFor`.
  Version-aware dump output (omitting `type_params=[]` on older
  versions, etc.) is wired in Round 1.5.4. For now the flag is
  accepted and exercised so the plumbing is tested.
- Grammar fixtures `102_version_walrus.py`, `103_version_general_deco.py`,
  `104_version_except_star.py`, `105_version_type_params.py`,
  `106_version_type_param_defaults.py` covering one representative
  construct per version boundary (3.8, 3.9, 3.11, 3.12, 3.13).

### Changed

- CLI version bumped to `0.3.2`.
