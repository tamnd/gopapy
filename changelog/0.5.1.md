# v0.5.1 — Unicode 15.1 and 16.0 printability fixes

## Fix

`pyRepr` calls `unicode.IsPrint` to decide whether to write a rune as
a literal or as a `\uXXXX` escape. Go ships Unicode 15.0 tables, where
eight codepoints are category Cn (unassigned, non-printable). Python
uses newer Unicode tables and treats those codepoints as printable,
causing `ast.dump()` to write them as literals while gopapy escaped them.

The split is version-dependent:

- Unicode 15.1 (Python 3.13): U+2FFC, U+2FFF, U+31EF became So.
- Unicode 16.0 (Python 3.14): U+1B4F, U+1B7F, U+1C89, U+2427, U+31E4
  became printable (Po, Lu, or So).

`pythonIsPrint` in `parser/ast_dump.go` now applies each group only
when the target minor version matches, keeping dumps byte-identical
across all supported Python versions (3.8 through 3.14).

## Test fixture

`tests/grammar/121_unicode_15_1_printable.py` contains all eight
codepoints in one string literal. The oracle verifies them as literals
(not escapes) when targeting Python 3.13 or 3.14, and as escapes when
targeting 3.8 through 3.12.

## Corpus

`pygments/unistring.py` removed from `MISMATCH_ALLOW` in
`internal/oracle/corpus_diff.sh`. The allow list is now empty, so the
array and the is-allowed check are gone. corpus-astdiff runs clean with
no allowed entries.
