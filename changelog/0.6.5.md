# v0.6.5: zero-copy string literals

Single-quoted strings without escape sequences now return a direct sub-slice of
the source buffer instead of copying bytes through `strings.Builder`. One
`IndexByte` call locates the closing quote; a second checks for backslashes.
When both succeed the token's `val` field is a Go string header pointing into
`src` — no heap allocation needed.

## Implementation

Two SIMD-accelerated `IndexByte` calls in `scanner.scanString` (lex.go):

```go
if !triple && !raw {
    if closeIdx := strings.IndexByte(s.src[s.off:], quote); closeIdx >= 0 {
        chunk := s.src[s.off : s.off+closeIdx]
        if strings.IndexByte(chunk, '\\') < 0 {
            s.col += closeIdx + 1
            s.off += closeIdx + 1
            return token{kind: tkString, val: chunk, pos: start}, nil
        }
    }
}
var b strings.Builder // slow path: escapes or triple-quoted
```

The fast path applies to plain `"..."` and `'...'` strings (and their `b`/`u`
prefixed variants). Triple-quoted and raw strings fall through to the Builder
path as before.

**Safety**: `val` is a sub-slice of the `src` argument passed to `ParseFile`.
`ParseFile` does not mutate `src` after scanning begins. Go strings are
immutable, so the sub-slice is safe to hold as long as the original `src`
string is reachable — which it always is while the caller holds the `*Module`.

## Throughput (Apple M4, 10 cores, in-memory benchmark)

Back-to-back serial comparison (5×20 s each), then 3×20 s parallel:

| | MB/s | vs CPython serial |
|-|------|------------------|
| v0.6.4 serial | 112.6 | 11.0x |
| v0.6.5 serial | 116.8 | 11.4x |
| v0.6.4 parallel | 371.8 | 36.4x |
| v0.6.5 parallel | 400.6 | 39.3x |
| CPython serial baseline | 10.2 | 1x |

+4% serial, +8% parallel. Allocation count: 2.35M → 2.0M per iteration (−15%).
Memory per iteration: 70 MB → 64 MB (−9%).

The allocation reduction comes directly from eliminating one `strings.Builder`
heap allocation per escape-free string token. The parallel gain reflects lower
per-goroutine GC pressure feeding back into coordination overhead.
