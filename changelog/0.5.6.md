# v0.5.6 — roundtrip exceptions documented as permanent

## Roundtrip allow list

Two files in `corpus/roundtrip.sh` have been in the allow list since v0.5.2
with a comment saying they are "tracked bugs." They are now documented as
permanent exceptions, not bugs to fix.

Both are Black test fixtures for Black-internal transformations, not user code.
The new parser passes corpus-astdiff for both; only the legacy unparser's output
differs from CPython's unparser:

- `pep_572_remove_parens.py` — walrus-operator paren removal. The legacy
  unparser drops parentheses around `(x := y)` in certain expression contexts
  where CPython's unparser keeps them.
- `preview_long_strings.py` — implicit string concatenation with unusual prefix
  combinations. The legacy unparser produces a different dump than CPython's
  unparser for these cases.

The allow list remains at 2. The roundtrip gate is strict for all other corpus
files. The comment in `corpus/roundtrip.sh` is updated accordingly.
