## [0.2.12] - 2026-04-28

Phase 1.A through 1.I of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

Stdlib parse: **1842 / 1842**. Zero failures for the first time.

### Fixed

- **Triple-quoted raw string backslash rule.** CPython's raw-string
  rule applies to both single- and triple-quoted raw strings: a
  backslash plus the following character is consumed as a literal
  two-byte unit. The scanner had a `!triple` guard that left the rule
  off for triple-quoted raw strings, causing `r"""\""" abc"""` and
  similar patterns in `idlelib/pyparse.py` to fail with "unexpected
  token ^".

- **F-string tuple expressions.** `f"{a, b, c}"` and `f"{name,}"` are
  valid Python — the comma produces a tuple as the interpolated value.
  The f-string expression parser was calling the single-expression
  path and reporting "trailing tokens". Changed `parseInterpExpr` to
  use `parseTestlistOrStarExpr` so commas are consumed. Fixes
  `idlelib/redirector.py` and `test/test_minidom.py`.

- **Comments inside f-string braces.** PEP 701 (Python 3.12+) allows
  `# ...` comments inside `{...}` in f-strings. The lexer was
  rejecting `#` with "comments not allowed". Added a comment-skip
  branch to `scanFTInterp`. Fixes `test/test_fstring.py`.

- **`\N{name}` escape in f-string body.** The Unicode name escape
  `\N{SNOWMAN}` inside an f-string body was being split at `{`, which
  the scanner then treated as an interpolation opener. Added a handler
  in `scanFTBody` that consumes `\N{...}` as a unit before `{` can
  trigger interpolation.

- **Match statement with parenthesised subjects.** `match ():` and
  `match (a, b):` both produced "unexpected token NEWLINE". Added a
  `parenFollowedByColon` scanner method that reads ahead to decide
  whether a `(` after `match` is the start of a parenthesised subject
  or a function call argument list. Fixes `test/test_global.py` and
  `test/test_patma.py`.

- **Match statement with soft-keyword subjects.** `match match:`,
  `match case:`, and `match type:` were not recognised as match
  statements because `looksLikeMatchStmt` rejected all reserved
  keywords. Added `isSoftKeyword(nxt.val)` to the allowed set.

- **With-statement single-expression paren.** `with (p / 'x').open()
  as f:` and `with (a if c else b) as v:` failed because the parser
  committed to the multi-context-manager interpretation after seeing
  `(`. After parsing the closing `)`, the parser now checks whether a
  postfix chain follows (`.`, `[`, or `(`) and if so re-interprets the
  parentheses as grouping for line continuation. Fixes
  `test/test_pathlib/test_pathlib.py` and `compileall.py`.

- **With-statement call-subscript `as` target.** `with ctx() as d[0]:`
  was accepted but `with ctx() as call()[0]:` was not because the
  `as` target parser used `parseTargetAtom` instead of `parseTrailer`.
  Changed to `parseTrailer` so any valid assignment target is
  accepted. Fixes `test/test_with.py`.

- **Raw f-string backslash rule.** In a raw f-string, a backslash
  followed by any character other than `{` or `}` is consumed as a
  two-byte literal unit. The `{` and `}` exceptions preserve their
  roles as interpolation delimiters. This makes `fr"\'"`, `fr"\\"`,
  and `fr"\\\'"` parse correctly. Fixes the remaining
  `test/test_fstring.py` cases.

- **F-string conversion trailing whitespace.** `f"{3!s  }"` was
  rejected with "expected '}' to close interpolation". The scanner
  now skips spaces and tabs after the conversion character before
  looking for `:` or `}`.

- **F-string self-documenting expression with trailing comment.**
  `f"{x = # note\n}"` was not identified as a self-doc expression
  because `parseSelfDoc` did not strip the trailing comment before
  looking for the `=` suffix. Added `stripLineComments` and
  `lastUnquotedHash` helpers. The stripped comment is also excluded
  from the `selfDocText` constant value, matching CPython's output.
