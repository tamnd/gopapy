# 0.2.6 - 2026-04-27

### Changed

- **Package layout flattened.** The new hand-written stack is now
  available under unsuffixed top-level paths and the original
  participle-based stack moves to `legacy/`:
  - `github.com/tamnd/gopapy/v2/parser2` → `github.com/tamnd/gopapy/parser`
  - `github.com/tamnd/gopapy/v2/linter2` → `github.com/tamnd/gopapy/linter`
  - `github.com/tamnd/gopapy/v2/symbols2` → `github.com/tamnd/gopapy/symbols`
  - `github.com/tamnd/gopapy/v2/diag` → `github.com/tamnd/gopapy/diag`
  - `github.com/tamnd/gopapy/parser` (participle) → `github.com/tamnd/gopapy/legacy/parser`
  - `github.com/tamnd/gopapy/linter` (v1) → `github.com/tamnd/gopapy/legacy/linter`
  - `github.com/tamnd/gopapy/symbols` (v1) → `github.com/tamnd/gopapy/legacy/symbols`
  - `github.com/tamnd/gopapy/diag` (v1) → `github.com/tamnd/gopapy/legacy/diag`
- **Package declarations renamed** to drop the `2` suffix:
  `package parser2` → `package parser`, `package linter2` → `package linter`,
  `package symbols2` → `package symbols`. Per Effective Go, numeric
  suffixes on package names are an anti-pattern; the rename + `legacy/`
  quarantine expresses the same generation split without polluting
  identifiers.
- **`v2/` directory removed.** All code now lives at the module root.
- Shared infrastructure (`ast`, `cst`, `lex`, `lsp`) stays at the top
  level and is consumed by both stacks.
- CLI version bumped to `0.2.6`.

### Migration

```diff
-import "github.com/tamnd/gopapy/v2/parser2"
+import "github.com/tamnd/gopapy/parser"

-import "github.com/tamnd/gopapy/v2/linter2"
+import "github.com/tamnd/gopapy/linter"

-import "github.com/tamnd/gopapy/v2/symbols2"
+import "github.com/tamnd/gopapy/symbols"
```

Qualified identifiers also drop the `2` suffix
(`parser2.Module` → `parser.Module`, etc.). Consumers still pinned to
the old participle stack should switch to the `legacy/...` paths.
