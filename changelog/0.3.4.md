## [0.3.4] - 2026-04-28

Phase 1.5, Round 4 of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

### Added

- **`TestOracle_Py314` — Python 3.14 oracle test (active, no skip).**
  `gopapy dump --py 3.14` is now verified byte-identical to CPython 3.14's
  `ast.dump()` for all 106 grammar fixtures. Runs alongside `TestOracle_Py313`
  (both active, neither skipped). Python 3.14 uses the same compact format as
  3.13 (omit empty fields), so this test passes without additional code changes.

- **Version-aware `ASTDump(m *Module, pyMinor int) string`.**
  The `ASTDump` function now accepts a Python minor version so callers can
  request version-specific dump output. The `gopapy dump --py 3.X` flag is
  wired through to `ASTDump`; the foundation for 3.8–3.12 compatibility is
  in place (see v0.3.5–v0.3.9 for the per-version implementations). The
  internal `dumper` struct carries `showEmpty bool` (true for ≤3.12) and
  `py38 bool` (true for ≤3.8) flags that control version-specific output.

- **`# Python 3.13+` marker on fixture 057 (`type_param_advanced.py`).**
  That fixture uses type parameter defaults (`[T = int]`, PEP 696) which
  require Python 3.13. The marker lets the oracle tests skip it gracefully
  under Python 3.12 and below.

### Changed

- CLI version bumped to `0.3.4`.
