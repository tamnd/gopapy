## [0.2.11] - 2026-04-27

Phase 1.4 + 1.5 + 1.6 of the parse-every-Python roadmap
(`~/notes/Spec/1200/1238_gopapy_every_python_roadmap.md`).

Three more soft-keyword sites and one lexer rule. Stdlib parse:
**1833 / 1842**, down from 28 failures (was 32).

### Fixed

- **Soft keywords as relative-import module names.** `from .case
  import TestCase` and friends were rejected with "expected 'import'
  in from-import" because the from-import module-name lookahead
  filtered out reserved keywords without distinguishing soft
  keywords. The `unittest` package (its `__init__.py`, `_log.py`,
  `async_case.py`, `runner.py`) all start with `from .case import
  ...` and now parse.
- **Soft keywords as walrus targets.** `if match := re.search(p, s):`
  is the standard idiom across the stdlib (`fractions`, `pdb`,
  `pydoc`, `tarfile`, `xmlrpc/server`, `test_android`,
  `test_named_expressions`) and was failing with "expected :, got
  :=". The walrus lookahead in `parseExpr` had the same
  reserved-vs-soft confusion as the call-args path. Same one-line
  fix.
- **Raw-string `\X` rule.** In a non-triple raw string, CPython
  treats backslash plus the following character as a literal
  two-byte unit — both bytes go into the value and the second byte
  never terminates the string. That is what makes `r'\''`,
  `r"\""`, `r"\\"`, and the `r'(\'[^\']*\')'` regex idiom valid.
  Our scanner had no raw-mode branch at all, so it terminated those
  strings at the first quote that followed a backslash and produced
  cascading "unexpected character `\\`" / "unexpected token ^"
  errors. Triple-quoted raw strings keep the existing rule (the
  triple-quote terminator wins; `r"""abc\"""` is still `abc\`).
  Hits `re/_parser.py`, `pydoc.py`, `_markupbase.py`,
  `xml/etree/ElementPath.py`, `idlelib/colorizer.py`, `platform.py`,
  `textwrap.py`, several `test/` files.

### Added

- `tests/grammar/090_softkw_more.py` covers `from .case import X`,
  `from .match import X`, `from .type import X`, and `match :=`,
  `case :=`, `(type := ...)` walrus targets.
- `tests/grammar/091_raw_string_quotes.py` covers `r'\''`, `r"\""`,
  `r"\\"`, the `r'(\'[^\']*\')'` regex idiom, the `r"\\Server\Share\\"`
  Windows-path idiom, and a triple-quoted raw string with backslashes
  to lock in the precedence between the two rules.
