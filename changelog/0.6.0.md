# v0.6.0: benchmark infrastructure

Adds measurement tools for the 0.6.x performance goal: parse the 12,179-file
corpus 100x faster than CPython ast.parse() on the same hardware.

## Baseline (Apple M4, 2026-04-28)

| Tool | MB/s | vs CPython |
|------|------|-----------|
| CPython 3.14 ast.parse (pre-loaded, single-threaded) | 10.2 | 1x |
| gopapy serial (pre-loaded, single-threaded) | 78 | 7.6x |
| gopapy parallel (pre-loaded, 10 goroutines) | 283 | 27.7x |
| Target | 1020 | 100x |

Gap to close: 3.6x improvement from serial goroutine baseline.
The subsequent 0.6.x releases each add one optimization.

## corpus/cpython_bench.py

New script. Pre-reads all corpus .py files into memory, runs ast.parse()
in a timed loop, and prints throughput in the same key: value format as
`gopapy bench`. Establishes the CPython baseline on any machine.

```
cpython-files: 12179
cpython-bytes: 127.7 MB
cpython-parse-rate: 971.4 files/s, 10.2 MB/s
```

## internal/bench/corpus_bench_test.go

New Go benchmark package. Pre-loads up to 4000 corpus files and runs
ParseFile in a loop with no disk I/O. Two variants:

- BenchmarkCorpusParseSerial: single goroutine, isolates per-optimization gain
- BenchmarkCorpusParseParallel: runtime.NumCPU() workers, shows parallel ceiling

Run: `go test -bench=BenchmarkCorpus -benchtime=5s -benchmem ./internal/bench/`

## CI bench job

Two steps added after the existing unit fixture benchmarks:

- `cpython ast.parse throughput (baseline)`: runs cpython_bench.py
- `gopapy corpus parse throughput`: runs BenchmarkCorpus with -benchtime=5s

Both are `continue-on-error: true`; they print numbers for every PR without
blocking the build.
