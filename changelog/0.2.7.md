# 0.2.7 - 2026-04-27

### Added

- **`parser.ASTDump`**: CPython-compatible `ast.dump()` for the
  hand-written parser's tree. The 1300-line implementation matches
  CPython 3.14 field order and quoting; verified against the
  `internal/oracle/oracle.py` driver across all 87 grammar fixtures.
- **PEP 654 `try/except*`**: parsed as a distinct `TryStar` AST node so
  the runtime can give it the correct group-handling semantics. Shares
  `ExceptHandler` with `Try`.
- **PEP 646 `*Ts` subscript**: a single starred subscript is wrapped in
  an implicit `Tuple` to match CPython.
- **Big-int literals**: integer literals that overflow `int64` now
  produce `*big.Int` Constant values instead of erroring. Float
  literals that overflow to `±Inf` or underflow to `0.0` are accepted
  (Python's behaviour). Adds `math/big` to the dependency closure.
- **PEP 701 self-documenting f-string `f"{x=}"`**: parsed as a
  `JoinedStr` with the literal source text injected before the
  `FormattedValue`, with default conversion `r` (repr) when no explicit
  conversion or format spec is given.
- **`u"..."` prefix**: scanned and round-tripped as a distinct token
  prefix so the surface syntax survives the parse.
- **Escape sequences**: `\a`, `\b`, `\f`, `\v`, `\xHH`, `\uHHHH`,
  `\UHHHHHHHH`, and 1–3 digit octal escapes are now decoded; previously
  only `\n`, `\t`, `\r`, `\\`, `\'`, `\"`, `\0`, and line-continuation
  were handled.
- Test fixtures `086_bigint.py` and `087_fstring_self_doc.py`.

### Changed

- **`dump` command** now routes through `parser.ParseFile` +
  `parser.ASTDump` instead of the legacy participle parser. The two
  produce identical output for the fixture corpus.
- **`check` command** now routes through `parser.ParseFile` instead of
  the legacy parser. Drops the participle-specific GC-pressure comment;
  the periodic GC is still useful for very large corpora.
- CLI version bumped to `0.2.7`.
