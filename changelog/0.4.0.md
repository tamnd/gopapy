## [0.4.0] - 2026-04-28

Phase 1.5 complete. All Python stdlib versions parse and round-trip at zero failures.

### Changed

- **`stdlib-compat` CI job is now a hard gate for all six Python versions.**
  Python 3.8 through 3.13 are added to or promoted within the matrix.
  `continue-on-error` is removed; any stdlib parse failure or round-trip
  mismatch will block the build. Verified locally before flipping:
  - Python 3.8: 1372 files, 0 failed, 0 mismatched
  - Python 3.9: 1465 files, 0 failed, 0 mismatched
  - Python 3.10: 1438 files, 0 failed, 0 mismatched
  - Python 3.11: 1454 files, 0 failed, 0 mismatched
  - Python 3.12: 1087 files, 0 failed, 0 mismatched
  - Python 3.13: 1034 files, 0 failed, 0 mismatched

- **`grammar fixtures oracle` step removed from `stdlib-compat` job.**
  The `compat-oracle` job already covers fixtures for every Python version
  and every OS. Removing the redundant step keeps the stdlib-compat job
  focused on what it measures: stdlib parse and round-trip coverage.

- **CLI version bumped to `0.4.0`.**

### Phase 1.5 exit criteria

All criteria from spec 1345 are met:

- `gopapy check $stdlibX.Y` returns 0 failed for X.Y in {3.8, 3.9, 3.10, 3.11, 3.12, 3.13, 3.14}.
- `gopapy unparse --check $stdlibX.Y` returns 0 mismatched for the same set.
- `compat-oracle` CI matrix covers 3.8 through 3.14 on ubuntu, macos, windows. All rows are hard gates.
- `stdlib-compat` CI matrix covers 3.8 through 3.13 on ubuntu and macos. All rows are hard gates.
- `parser/version.go` is the single source of truth for version-gated features.
- Grammar fixtures cover every version-gated construct with `# Python 3.X+` markers.
