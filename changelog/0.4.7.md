# v0.4.7 — corpus-astdiff script: skip mutually-invalid files

## Fixes

- **`corpus_diff.sh` BOTH_ERROR classification**: When both gopapy and CPython
  fail to parse a file, the script now classifies it as `BOTH_ERROR` and counts
  it as `skip` rather than `fail`. Previously, the gopapy error was caught first
  and counted as `GOPAPY_ERROR`, hiding the fact that CPython also rejected the
  file. Seven such files in the sample were intentionally-invalid Python
  (black test fixtures for async-as-identifier, invalid pattern matching,
  invalid headers, Python 2 syntax, PEP 572 invalid parens, PEP 701 invalid
  f-strings, and Django syntax error tests).

- **`corpus_diff.sh` CPYTHON_ERROR reclassified as skip**: When gopapy succeeds
  but CPython fails (e.g. `lxml/selftest.py` which has an encoding issue in
  CPython's `open()`), the result is now counted as `skip` rather than `fail`,
  since the divergence is not a gopapy bug.

- **Summary line includes skip count**: The final output now reads
  `pass=N skip=N fail=N` so intentionally-skipped files are visible.

## Result

corpus-astdiff (seed=99, 9011 files): **pass=9001 skip=8 fail=2**

The 2 remaining mismatches are known limitations:
- `tricky_unicode_symbols.py`: NFKC identifier normalization (µ→μ) requires
  `golang.org/x/text`, deferred to a later release.
- `pygments/unistring.py`: Go Unicode 15.0 vs Python Unicode 15.1 character
  category disagreements (e.g. U+1B4F, U+1B7F).
