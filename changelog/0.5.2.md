# v0.5.2 — legacy-parser roundtrip fixes

## Fixes

Four parse failures in the legacy parser (used by `gopapy unparse`) are
resolved. `corpus/roundtrip.sh` now allows 2 files instead of 8.

### Annotated-assignment tuple RHS

`x: T = a, b` is valid Python: the annotation and assignment are
separate, and the RHS may be any star_expressions (including a bare
tuple). The legacy grammar had `AnnVal *Expression`, which rejected
the comma. Changed to `AnnVal *AssignTarget` so comma-separated RHS
values are accepted and emitted as a Tuple node.

Three corpus files benefited: `pillow/TiffImagePlugin.py`,
`pillow/WmfImagePlugin.py`, and `black/python38.py`.

### Form feeds in source

CPython's tokenizer treats form feed (`\x0c`) in the leading whitespace
of a line as a column-counter reset: indentation is measured from the
last form feed, not from the start of the line. Our scanner counted form
feeds as one byte of column width, so `\x0cdef` appeared at column 1
instead of column 0, causing the indent processor to emit a spurious
INDENT.

Added `atLineStart bool` to `lex.Scanner`. Form feeds advance the byte
offset (so error positions are correct) but reset the column to 0 when
`atLineStart` is true. The flag is set after each `\n` and cleared when
the first non-whitespace token of a line is produced.

### Type param defaults with `*`

PEP 696 allows TypeVarTuple defaults like `*Ts = *int`. The `*` before
the default expression was rejected as unexpected. Added `DefaultStar
bool` to the `TypeParam` grammar node and updated `emitTypeParams` in
`ast/emit.go` to wrap the default in a `Starred` node when `DefaultStar`
is set.

### Other_ID_Continue in the `lex` package

`isOtherIDContinue` existed only in `parser/lex.go` (the new parser).
The `lex` package (`lex/scanner.go`) used by the legacy parser was
missing the same set of UAX #31 codepoints (U+00B7, U+0387, U+1369–
U+1371, U+19DA, U+E0100–U+E01EF). Added `isOtherIDContinue` to
`lex/scanner.go`; `isIdentPart` now calls it. Fixes `A᧚` (U+19DA) and
`A፩` (U+1369) identifiers in `black/tricky_unicode_symbols.py`.

## Corpus

`corpus/roundtrip.sh` ALLOW list reduced from 8 entries to 2. The two
remaining allows are `pep_572_remove_parens.py` and
`preview_long_strings.py`, both require unparser-level fixes deferred
to a later release.
