## [0.4.1] - 2026-04-28

Phase 2 Round 1 of the parse-every-Python roadmap.

### Added

- **Corpus harness scaffold (`corpus/`).**
  Three shell scripts and a pinned package list make up the corpus harness.
  No Go code changes in this release; the harness is shell-only.

  - `corpus/packages.txt` — 18 PyPI packages pinned to April 2026 stable
    releases: django, flask, requests, numpy, pandas, sqlalchemy, black,
    mypy, pytest, setuptools, pip, pyyaml, boto3, cryptography, lxml,
    pillow, tornado, celery.

  - `corpus/download.sh` — downloads source distributions (sdists) for
    every package in packages.txt using `pip download --no-binary :all:`,
    then extracts all `.py` files into `corpus/src/`. Idempotent: skips
    the download if `corpus/src/` is already populated.

  - `corpus/check.sh` — runs `gopapy check corpus/src/` and reports pass
    and fail counts. Exits non-zero if any file fails to parse.

  - `corpus/roundtrip.sh` — runs `gopapy unparse --check corpus/src/`
    and reports round-trip mismatch counts. Exits non-zero on any mismatch.

- **`.gitignore` entries for corpus cache directories.**
  `corpus/sdists/` and `corpus/src/` are ignored so the downloaded
  packages do not end up in version control.

### Changed

- CLI version bumped to `0.4.1`.
