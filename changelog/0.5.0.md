# v0.5.0 — NFKC identifier normalization

## Fix

Python normalizes identifiers to NFKC before storing them in the AST.
The source text `µ` (U+00B5 MICRO SIGN) and the source text `μ`
(U+03BC GREEK SMALL LETTER MU) refer to the same identifier: both
tokenize to the NFKC form `μ`. gopapy previously returned identifiers
as raw source bytes, so `µ` stayed `µ` in the AST dump rather than
becoming `μ`, causing a mismatch against CPython's `ast.dump()`.

`parser/lex.go` now applies `norm.NFKC.String` in
`scanNameOrPrefixedString` after scanning an identifier. Pure-ASCII
names skip normalization via a fast byte scan (the common case).

Dependency added: `golang.org/x/text v0.36.0`.

## Test fixture

`tests/grammar/120_nfkc_identifier.py` covers the µ -> μ case and
is verified by the oracle against Python 3.8 through 3.14.

## Corpus
