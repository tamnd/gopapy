# v0.5.5 — gocopy wiring verified

## API compatibility

Confirmed that all gopapy symbols used by gocopy v0.4.5 are present and
unchanged in v0.5.5. The symbols verified:

- `parser.ParseFile` — primary parse entry point
- `ast.Assign`, `ast.AnnAssign`, `ast.AugAssign`
- `ast.BinOp`, `ast.UnaryOp`, `ast.BoolOp`
- `ast.Call`, `ast.Attribute`, `ast.Subscript`
- `ast.FunctionDef`, `ast.AsyncFunctionDef`, `ast.ClassDef`
- `ast.Import`, `ast.ImportFrom`, `ast.alias`
- `ast.If`, `ast.For`, `ast.While`, `ast.With`, `ast.withitem`
- `ast.Return`, `ast.Raise`, `ast.Try`, `ast.ExceptHandler`
- `ast.Constant`, `ast.Name`, `ast.Tuple`, `ast.List`, `ast.Dict`
- `ast.Load`, `ast.Store`, `ast.Del`
- `diag.Error`

No API breaks between v0.4.1 (what gocopy currently imports) and v0.5.5.

## Dependency note

gocopy upgrading from gopapy v0.4.1 to v0.5.5 will gain one new indirect
dependency: `golang.org/x/text v0.36.0` (used by the NFKC identifier
normalizer added in v0.5.0). Run `go mod tidy` after `go get` to pick it up.

## No code changes

This release ships no parser or API changes. It is a version bump that marks
the point at which gocopy can safely upgrade.
