# v0.5.4 — corpus-stats CI step

## New: corpus stats

`corpus/stats.sh` prints a grep-friendly health summary for the corpus.
The script runs automatically as a step in the `corpus-parse` CI job after
the parse gate, so every PR includes throughput and allow-list numbers in
the run log.

Sample output:

```
corpus-files: 12179
corpus-bytes: 121.8 MB
corpus-parse-rate: 5954.7 files/s, 59.6 MB/s
corpus-roundtrip-allowed: 2
corpus-astdiff-pass: 100
corpus-astdiff-skip: 0
corpus-astdiff-fail: 0
```

## Updated: gopapy bench

`gopapy bench DIR` now uses the new parser (`parser.ParseFile`) instead of
the legacy participle parser. Output keys are prefixed with `corpus-` to
match the stats format and be easier to grep across runs:

```
corpus-files: N
corpus-bytes: X.X MB
corpus-parse-rate: X.X files/s, X.X MB/s
```
