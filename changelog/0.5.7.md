# v0.5.7 — housekeeping

## Fix: lspCmd unused stderr parameter

`cmd/gopapy/main.go`: the `lspCmd` function accepted a `stderr io.Writer`
parameter that was never used. LSP errors go through the JSON-RPC protocol
on stdout; there is no direct stderr output from the LSP server. The parameter
is now named `_`.

## Fix: stale compat-oracle comments

Two comments described the version-aware ASTDump as a future / in-progress
item. It was completed well before v0.5.x. Both are updated to reflect the
current state:

- `.github/workflows/ci.yml` compat-oracle job comment: removed the note
  saying "3.12 and below are skipped until version-aware dump is implemented."
  The oracle tests for Python 3.8–3.14 are live and gate CI.
- `tests/compat/compat_test.go` package comment: replaced the "implemented
  incrementally (see spec 1321-1324)" note and the "3.12 and below skipped"
  paragraph with an accurate description of how `showEmpty` (pyMinor ≤ 12)
  and `py38` (pyMinor ≤ 8) work.

## Change: corpus-astdiff sample 100 → 500

The corpus has 12179 .py files. Sampling 100 per CI run was 0.8% coverage;
the full corpus would rotate in about 122 days. At 500 files per run the
coverage is 4.1% and the full corpus rotates in about 25 days.

The step in `ci.yml` now passes 500 to `corpus_diff.sh`. The `corpus/stats.sh`
quick-report step stays at 100 for speed.
